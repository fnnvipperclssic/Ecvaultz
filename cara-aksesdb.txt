================================================================================
                    CARA MENGAKSES DATABASE ECVAULTZ
                          (Docker - MariaDB)
================================================================================

A. AKSES DARI TERMINAL CMD (Windows)

   1. Masuk ke container MariaDB:
      docker exec -it ecvaultz-mariadb bash

   2. Setelah masuk container, jalankan MariaDB client:
      mariadb -u root -prootpassword ecvaultz

   3. Atau langsung dari host tanpa masuk container:
      docker exec -it ecvaultz-mariadb mariadb -u root -prootpassword ecvaultz

   4. Bisa juga via PowerShell:
      docker exec -it ecvaultz-mariadb mariadb -u root -prootpassword ecvaultz


B. PERINTAH SQL YANG SERING DIGUNAKAN

   SHOW TABLES;

   DESCRIBE users;
   DESCRIBE files;
   DESCRIBE file_shares;
   DESCRIBE folders;
   DESCRIBE activity_logs;

   SELECT id, name, email, is_admin, two_factor_enabled,
          email_verified_at, last_login_at, created_at
   FROM users;

   SELECT id, uuid, user_id, original_name, size, mime_type,
          is_encrypted, download_count, uploaded_at, deleted_at
   FROM files;

   SELECT f.original_name, f.size, f.uploaded_at
   FROM files f JOIN users u ON f.user_id = u.id
   WHERE u.email = 'i.thaufanm.stu@pnc.ac.id';

   SELECT fs.uuid, f.original_name, fs.type, fs.permission,
          fs.external_email, fs.share_link_token, fs.expires_at,
          fs.access_count
   FROM file_shares fs JOIN files f ON fs.file_id = f.id;

   SELECT al.id, u.name, al.action, al.ip_address, al.created_at
   FROM activity_logs al LEFT JOIN users u ON al.user_id = u.id
   ORDER BY al.created_at DESC LIMIT 20;

   SELECT id, user_id, ip_address, last_activity
   FROM sessions WHERE user_id IS NOT NULL;

   SELECT
     (SELECT COUNT(*) FROM users) AS total_users,
     (SELECT COUNT(*) FROM files) AS total_files,
     (SELECT COUNT(*) FROM files WHERE deleted_at IS NOT NULL) AS trashed_files,
     (SELECT COUNT(*) FROM file_shares) AS total_shares,
     (SELECT COUNT(*) FROM activity_logs) AS total_logs;


C. BACKUP & RESTORE DATABASE

   Backup:
   docker exec ecvaultz-mariadb mariadb-dump -u root -prootpassword ecvaultz > backup.sql

   Restore:
   docker exec -i ecvaultz-mariadb mariadb -u root -prootpassword ecvaultz < backup.sql


D. KONEKSI DARI APLIKASI EKSTERNAL (DBeaver / HeidiSQL / TablePlus)

   Host     : localhost
   Port     : 3306
   User     : root
   Password : rootpassword
   Database : ecvaultz
   Driver   : MySQL / MariaDB


E. CATATAN PENTING

   - User root adalah untuk DEVELOPMENT saja
   - Untuk production, GANTI password dan gunakan dedicated user
   - Data persisten di Docker volume "mariadb_data"
   - File fisik user disimpan di: storage/app/private/
